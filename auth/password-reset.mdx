---
title: "Password reset"
description: "Request a time-limited OTP by email and use it to set a new password."
---

Password reset is a two-step flow. Both endpoints are **public** — no session cookie is required.

<Warning>
  The account must have a **verified email** before a password reset is allowed. Unverified accounts receive an error directing them to contact an administrator.
</Warning>

<Steps>
  <Step title="Request a reset OTP">
    Call `POST /api/auth/send-reset-otp` with your email address. The server generates a 6-digit OTP, stores it with a **15-minute expiry**, and sends it to the registered email.
  </Step>
  <Step title="Reset the password">
    Call `POST /api/auth/reset-password` with your email, the OTP, and the new password. On success, the password is updated (bcrypt, 10 rounds) and the stored OTP is cleared.
  </Step>
</Steps>

---

## Step 1 — Send reset OTP

`POST /api/auth/send-reset-otp` — public

### Request body

<ParamField body="email" type="string" required>
  The email address associated with the account.
</ParamField>

### Responses

<CodeGroup>
```json Success
{
  "success": true,
  "message": "OTP sent to your email"
}
```

```json Account not verified
{
  "success": false,
  "message": "Cuenta no verificada. Por favor contacta al administrador para restablecer tu contraseña."
}
```

```json User not found
{
  "success": false,
  "message": "User not found"
}
```
</CodeGroup>

### curl example

```bash
curl -X POST https://api.example.com/api/auth/send-reset-otp \
  -H "Content-Type: application/json" \
  -d '{"email": "jane@example.com"}'
```

---

## Step 2 — Reset password

`POST /api/auth/reset-password` — public

### Request body

<ParamField body="email" type="string" required>
  The email address associated with the account.
</ParamField>

<ParamField body="otp" type="string" required>
  The 6-digit OTP received by email.
</ParamField>

<ParamField body="newPassword" type="string" required>
  The new plain-text password. Hashed with bcrypt (10 rounds) before storage.
</ParamField>

### Responses

<CodeGroup>
```json Success
{
  "success": true,
  "message": "Password has been reset successfully"
}
```

```json Missing fields
{
  "success": false,
  "message": "Email, OTP, and new password are required"
}
```

```json Invalid OTP
{
  "success": false,
  "message": "Invalid OTP"
}
```

```json Expired OTP
{
  "success": false,
  "message": "OTP Expired"
}
```

```json Account not verified
{
  "success": false,
  "message": "Cuenta no verificada. Acción denegada."
}
```
</CodeGroup>

### curl example

```bash
curl -X POST https://api.example.com/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jane@example.com",
    "otp": "739104",
    "newPassword": "newS3cur3P@ss"
  }'
```

---

## OTP details

| Property | Value |
|---|---|
| Format | 6-digit numeric string |
| Expiry | **15 minutes** from the time of sending |
| Invalidation | Cleared from the database after a successful reset |

<Warning>
  The reset OTP expires after only **15 minutes**. If it expires before you submit it, request a new one by calling `send-reset-otp` again.
</Warning>
