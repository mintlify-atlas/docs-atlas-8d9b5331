---
title: "Quickstart"
description: "Get QuickFind API running locally and make your first authenticated request in minutes."
---

<Steps>
  <Step title="Clone the repository">
    Clone the repository and navigate into the project directory.

    ```bash
    git clone <your-repo-url>
    cd <repo-directory>
    ```
  </Step>

  <Step title="Install dependencies">
    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Copy the example file and fill in your values.

    ```bash
    cp .env.example .env.development
    ```

    At minimum, set the following variables before starting the server:

    ```bash .env.development
    PORT=4000
    NODE_ENV=development

    JWT_SECRET=replace_with_a_strong_secret

    DB_HOST=localhost
    DB_PORT=5432
    DB_NAME=postgres
    DB_USER=your_db_user
    DB_PASSWORD=your_db_password
    DB_SSL=false
    DB_SSL_REJECT_UNAUTHORIZED=false

    SMTP_USER=your_smtp_user
    SMTP_PASS=your_smtp_password
    SENDER_EMAIL=no-reply@yourdomain.com
    ```

    See [Environment setup](/environment-setup) for a description of every variable.
  </Step>

  <Step title="Start the server">
    ```bash
    npm start
    ```

    The server waits for a successful PostgreSQL connection before binding to the port. A successful startup prints output similar to:

    ```
    ✅ Database Connected
    ✅ Modelos sincronizados
    2026-03-25 10:00:00 [INFO]: 🚀 Server started on PORT:4000
    ```

    <Tip>Use `npm run server` during development to get automatic restarts via nodemon.</Tip>
  </Step>

  <Step title="Make your first request">
    ### Register a user

    ```bash
    curl -X POST http://localhost:4000/api/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "email": "jane@example.com",
        "password": "Str0ng!Pass",
        "name": "Jane",
        "surname": "Doe",
        "country": "US",
        "birth_date": "1990-05-15"
      }'
    ```

    ### Log in

    ```bash
    curl -c cookies.txt -X POST http://localhost:4000/api/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "email": "jane@example.com",
        "password": "Str0ng!Pass"
      }'
    ```

    A successful login returns:

    ```json
    {
      "success": true,
      "role": 1
    }
    ```

    The JWT token is set automatically in an `HttpOnly` cookie named `token`.

    <Note>
      Cookie-based auth requires a cookie jar when using curl. Pass `-c cookies.txt` to save cookies on login and `-b cookies.txt` to send them on subsequent requests.
    </Note>

    ### Make an authenticated request

    ```bash
    curl -b cookies.txt http://localhost:4000/api/auth/is-auth
    ```
  </Step>
</Steps>
