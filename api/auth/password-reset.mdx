---
title: "Password reset"
description: "Request a password-reset OTP and set a new password"
---

Password reset is a two-step flow:

1. **Send OTP** — request a 6-digit reset code to be emailed to the account address.
2. **Reset password** — submit the code along with the new password to complete the reset.

Neither endpoint requires an active session cookie.

<Info>
  The account must be verified (`is_account_verified = true`) before a password reset can be initiated. Unverified accounts must contact an administrator.
</Info>

---

## Step 1 — Send reset OTP

`POST /api/auth/send-reset-otp`

Looks up the account by email, generates a 6-digit OTP with a **15-minute expiry**, and sends it to the registered email address.

### Request body

<ParamField body="email" type="string" required>
  The email address of the account for which the password should be reset.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  `true` when the OTP was sent, `false` otherwise.
</ResponseField>

<ResponseField name="message" type="string">
  Status message.
</ResponseField>

### Example

```bash
curl -X POST http://localhost:4000/api/auth/send-reset-otp \
  -H "Content-Type: application/json" \
  -d '{"email": "jane@example.com"}'
```

#### Success response

```json
{
  "success": true,
  "message": "OTP sent to your email"
}
```

#### Error responses

```json
{
  "success": false,
  "message": "Email is required"
}
```

```json
{
  "success": false,
  "message": "User not found"
}
```

```json
{
  "success": false,
  "message": "Cuenta no verificada. Por favor contacta al administrador para restablecer tu contraseña."
}
```

### Error codes

| HTTP status | `success` | `message` | Cause |
|-------------|-----------|-----------|-------|
| 200 | `true` | `OTP sent to your email` | OTP generated and emailed |
| 200 | `false` | `Email is required` | `email` field was missing from the request body |
| 200 | `false` | `User not found` | No account exists for the given email |
| 200 | `false` | `Cuenta no verificada...` | Account exists but `is_account_verified` is `false` |

## Step 2 — Reset password

`POST /api/auth/reset-password`

Verifies the OTP submitted against the stored value and expiry timestamp. If valid, hashes and saves the new password, then invalidates the OTP.

### Request body

<ParamField body="email" type="string" required>
  The email address of the account being reset.
</ParamField>

<ParamField body="otp" type="string" required>
  The 6-digit reset code received by email. Valid for **15 minutes** after the send request.
</ParamField>

<ParamField body="newPassword" type="string" required>
  The new password to set for the account. Stored as a bcrypt hash.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  `true` when the password was reset, `false` otherwise.
</ResponseField>

<ResponseField name="message" type="string">
  Status message.
</ResponseField>

### Example

```bash
curl -X POST http://localhost:4000/api/auth/reset-password \
  -H "Content-Type: application/json" \
  -d '{
    "email": "jane@example.com",
    "otp": "371842",
    "newPassword": "N3wS3cur3P@ss"
  }'
```

#### Success response

```json
{
  "success": true,
  "message": "Password has been reset successfully"
}
```

#### Error responses

```json
{
  "success": false,
  "message": "Email, OTP, and new password are required"
}
```

```json
{
  "success": false,
  "message": "Invalid OTP"
}
```

```json
{
  "success": false,
  "message": "OTP Expired"
}
```

### Error codes

| HTTP status | `success` | `message` | Cause |
|-------------|-----------|-----------|-------|
| 200 | `true` | `Password has been reset successfully` | OTP matched, password updated |
| 200 | `false` | `Email, OTP, and new password are required` | One or more required fields were missing |
| 200 | `false` | `User not found` | No account exists for the given email |
| 200 | `false` | `Invalid OTP` | Submitted OTP does not match the stored value |
| 200 | `false` | `OTP Expired` | OTP was valid but the 15-minute window has passed |

<Warning>
  The reset OTP expires after **15 minutes**. If the code expires, start the flow again by calling `POST /api/auth/send-reset-otp`.
</Warning>
