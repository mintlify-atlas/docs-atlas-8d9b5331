## What is QuickFind API?

QuickFind API is a secure backend service that handles the full authentication lifecycle and user management for the QuickFind platform. It provides:

- **Authentication** — Register, login, logout with JWT tokens stored in HttpOnly cookies
- **Email verification** — OTP-based account verification via SMTP/Brevo
- **Password reset** — Time-limited OTP (15-minute expiry) for secure password recovery
- **User management** — Admin-only CRUD operations on user accounts
- **Role-based access control** — Middleware-enforced permissions by role ID
- **Area classification** — Level/area assignments for organizational hierarchy

## Base URL

```
http://localhost:4000
```

In production, replace with your deployed server address (e.g., AWS EC2 or load-balanced endpoint behind NGINX).

## API overview

| Resource | Base path | Description |
|---|---|---|
| Auth | `/api/auth` | Register, login, logout, email verification, password reset |
| Users | `/api/user` | Profile retrieval and admin user management |
| Roles | `/api/roles` | Role CRUD operations |
| Level Areas | `/api/levelArea` | Area/level classification CRUD |

<Note>
All authenticated routes require a valid JWT token stored in the `token` cookie. Include `credentials: 'include'` (fetch) or `withCredentials: true` (Axios) in all cross-origin client requests.
</Note>
